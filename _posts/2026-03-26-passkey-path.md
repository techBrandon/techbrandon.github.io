Passkeys are replacing passwords - but what that means depends on who you are. An IT admin configuring Entra policies, an end user setting up their first passkey, a security lead weighing downgrade attack risk, and a helpdesk agent fielding recovery calls all need different answers.

**[Passkey Path](/passkey-path/)** is an interactive guide organized by role. Pick yours, follow the path, and dig into the topics that matter to your job.

## Four paths

- **IT Admin** - Configure Entra authentication methods, plan phased rollouts, manage attestation and AAGUID allowlists, build Conditional Access policies
- **End User** - Understand what passkeys are, set them up on your devices, know what to do if something goes wrong
- **Security Lead** - Assess risk, understand downgrade attacks, build a phishing-resistant MFA strategy, map to compliance frameworks
- **Helpdesk** - Handle device loss recovery, issue Temporary Access Passes, verify identity, troubleshoot common issues

Every page links to related content across all four tracks, so you're never stuck in a single lane.
